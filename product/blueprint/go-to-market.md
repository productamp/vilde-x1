# Go-To-Market Blueprint

## Objective

Drive low-friction acquisition by giving small businesses immediate value before purchase.

## Method 1: Free Templates (GitHub + MIT)

Provide free, production-ready website templates for common small business types, for example:
- Restaurants
- Yoga studios
- Local service businesses
- Clinics/wellness practices

Distribution model:
- Publish templates in public GitHub repositories.
- License templates under MIT.
- Make cloning and remixing simple for non-technical users.

## Why this works

- Removes blank-page friction and gives immediate first value.
- Expands top-of-funnel discovery through GitHub search and sharing.
- Builds trust by making quality artifacts publicly accessible.
- Creates a natural upgrade path from free template usage to ProductVibe workflows.

## Product funnel connection

1. User discovers a template on GitHub.
2. User clones/starts quickly with minimal setup.
3. User uses ProductVibe to customize website + CMS content.
4. User publishes and iterates.
5. User upgrades to Pro when they need advanced workflows (for example Autopilot, Team Sync, Analytics depth).

## Operational notes

- Keep templates opinionated and industry-specific.
- Include clear README setup steps and preview screenshots.
- Maintain consistent quality bar across all free template repos.
- Track which template categories drive the highest activation and upgrade conversion.
