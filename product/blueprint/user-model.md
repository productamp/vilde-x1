# User Model

## Core Segments

### Free User (Normal User)

- Goal: get value fast without setup friction.
- Access: no login required.
- Experience: simple, elegant, low-complexity workflows.
- Connectivity expectation: app should work even when not always online for core flows.

### Pro User (Power User)

- Goal: run advanced workflows and higher-volume execution.
- Access: login required.
- Reason for login: payment protection, entitlements, secure backend workflows, and account-level continuity.
- Experience: deeper controls and premium capabilities.

## Access Principles

- Free should be instant-access and anonymous by default.
- Login prompts should appear only when a user upgrades or activates Pro-only features.
- The app is always the primary interface for user actions.
- Backend portal services handle authenticated execution for Pro functionality.

## Pro-Only Backend Workloads (Examples)

- Content Autopilot scheduling and workflow runs
- Team Sync collaboration state
- Payment-protected premium services and account-level settings

## Product Bet

Lower friction at entry drives adoption; depth and protected power workflows drive paid conversion.
