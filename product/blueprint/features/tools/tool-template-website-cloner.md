# Tool — Template and Website Cloner

Convert external templates and live websites into reusable ProductVibe templates.

## Purpose

Create a repeatable method to clone high-quality references and normalize them into internal templates that users can start from quickly.

## Input sources

- Public template URLs
- Live website URLs
- Public GitHub repositories

## Core workflow

1. Ingest source (URL or repo).
2. Capture structure and visual references (layout, sections, style patterns).
3. Convert to ProductVibe template format (componentized sections + editable content fields).
4. Clean and normalize content (remove copied brand content, keep neutral placeholders).
5. Validate template quality and responsiveness.
6. Save to internal template library.

## Output

- A reusable ProductVibe template package.
- Category tags (for example: restaurant, yoga studio, local service).
- Preview thumbnail and short description for template gallery.

## Guardrails

- Do not copy proprietary branding, logos, or private assets.
- Respect licensing and usage rights from source materials.
- Ensure generated template content is generic and safe for reuse.
- Apply a consistent quality standard before publication.

## Goal

Build a scalable pipeline for creating strong starter templates from real-world references, improving activation and template quality across the product.
