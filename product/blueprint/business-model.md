# Business Model

## Tier Structure

### Free Tier

- Includes up to **400 messages per month**.
- Designed as the entry tier (aligned to the Lovable $100 plan benchmark).
- **No login required**.
- Intended for normal users who want immediate usage without account setup.

### Pro Tier

- **Unlimited messages** after the free-tier limit.
- Unlocks premium features for more advanced users.
- **Login required** to protect payments, entitlements, and cloud workflows.
- Intended for power users running advanced operational workflows.

## Pro Features

- Sitemap
- Translations
- Design
- Designer Quality Output (Impeccable-style design polish)
- Team Sync (GitHub sync + sharing)
- Data Analytics (guided integration + on-demand refresh pulls)
- Content Autopilot (portal-backed scheduling with login/auth and app-first control for blog-first CMS generation/adaptation)

## Access Model

- Free access is anonymous/low-friction: users can start and use the app without creating an account.
- Pro access is authenticated: account login is required for billing, feature entitlements, and secure backend operations.
- The app remains the main interface in both tiers; backend portal services run behind the scenes for Pro workflows.

## Offline/Connectivity Principle

- Product direction is download-first and not always-online.
- Core app workflows should stay usable offline where possible.
- Online connectivity is required when Pro services need backend execution (for example scheduling, sync, or payment-protected actions).

## Positioning

- Keep onboarding and first value simple with free usage.
- Use premium feature unlocks + unlimited usage as the main Pro upgrade path.
